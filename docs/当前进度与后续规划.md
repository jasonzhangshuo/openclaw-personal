## 一、项目目标（蓝图核心）

建立一套**可持续演进的 Agentic 工作底座**：

- **1 个 OpenClaw Gateway** + **1 个飞书机器人** + **多个飞书群**
- **bindings** 把不同群路由到不同 Agent
- **不同 Agent 用不同 LLM**（成本/能力分层）
- **MemOS** 做外置记忆：可控共享、不串味、成本优化

用户场景：一个 bot、多个群，进不同群自动换角色/模型，省成本且可长期演进。

### 重要结论（避免再踩坑）

1. **同一飞书 app 只能被一个 OpenClaw 连接**：若云服务器/别处也曾用同一 app，需在那边停掉 Gateway，否则消息会被推送到那边，本机可能收到 429（例如那边用了 DeepSeek 被限流）。
2. **本机 Gateway**：由 launchd 常驻，config 用项目内 `.openclaw/config`（OPENCLAW_CONFIG_PATH）；API Key 来自 `.openclaw/state/.env`（脚本会把项目根 `.env` 拷过去）。
3. **飞书开放平台**：事件订阅用长连接；需至少 `im.message.receive_v1`；若需解析发送者名字，需开通 `contact:contact.base:readonly` 等权限。

## 六、M2 划分与国产模型建议（设计参考）

### 6.1 OpenClaw 里常见的 Agent 划分方式

社区和文档里常见两类思路，可以组合用：

| 思路 | 典型划分 | 适用场景 |
|------|----------|----------|
| **按职能/场景** | 脑暴 / 写作 / Coding / 资讯 + main 兜底 | 一个 bot 多群，不同群干不同事；和本项目蓝图一致。 |
| **按边界/隔离** | 工作 / 个人 / 实验沙箱；或多客户各一 Agent | 上下文不串、隐私隔离、实验不影响生产。 |

本项目是「**1 个飞书 bot + 多群**」，优先用**按职能划分**；若以后有「工作群 vs 生活群」需求，再按边界加 Agent 即可。

### 6.2 推荐 M2 划分（与蓝图一致）

- **main**：兜底（私信、未绑定群、临时拉群），建议用**主力/通用**模型，稳定、成本可控。
- **brainstorm**：脑暴/讨论/点子，偏创意、短回复多，可用**便宜或带免费额度**的模型。
- **writer**：写作/长文/润色，需要**长上下文、文风稳**的模型。
- **coding**：代码/实现/排查，需要**代码能力强、可联网或工具**的模型。
- **资讯**（可选）：摘报/日报/推送，可和 main 共用模型或单独省成本模型。

bindings 顺序：先「群 ID → 具体 agent」，最后一条「channel + accountId」无 peer，兜底到 main。

### 6.2.1 当前实现（与上文设计参考的对应）

| Agent | 群 / 场景 | 模型 | 密钥 | 提示词 |
|-------|-----------|------|------|--------|
| **main** | 私信、未绑定群 | `zai/glm-5` | ZAI_API_KEY | workspace-main 的 SOUL/AGENTS |
| **brainstorm** | 脑暴群 `oc_f566...` | `deepseek/deepseek-chat` | DEEPSEEK_API_KEY | workspace-brainstorm/SOUL.md（脑暴/讨论人设） |
| **coding** | 编程群 `oc_ce2c...` | `zai/glm-5` | ZAI_API_KEY | workspace-coding/SOUL.md（编程/实现人设） |

不同群需不同行为时，在对应 workspace 的 **SOUL.md**（及可选 AGENTS.md）里写清人设与规则即可，无需重启。

### 6.3 国产模型分层与各 Agent 建议

（以下 provider/model 以 OpenClaw 内置或 ClawCN 国产模型指南为准；密钥用 `.env` + 脚本同步到 `.openclaw/state/.env`。）

| 用途 | 推荐模型 | Provider | 说明 |
|------|----------|----------|------|
| **主力/兜底 (main)** | DeepSeek / 智谱 GLM | `deepseek/deepseek-chat`、`zai/glm-4.7` 或 `zai/glm-5` | 性价比高、中文好；你已在用 `zai/glm-5` 可继续当 main。 |
| **省成本/脑暴 (brainstorm)** | 通义千问 | `qwen-portal/coder-model` | OAuth 免 Key，每日约 2000 次免费，适合轻量讨论。 |
| **写作 (writer)** | 智谱 GLM / Kimi | `zai/glm-4.7`、`moonshot/kimi-k2.5` | 中文表达好；Kimi 长上下文适合长文。 |
| **Coding (coding)** | DeepSeek / MiniMax | `deepseek/deepseek-chat`、`minimax/MiniMax-M2.1` | 代码能力强；MiniMax 有 Coding Plan OAuth 可免 Key。 |

**简要建议**：

- **当前已采用**：brainstorm 用 DeepSeek、coding 用智谱 zai，main 兜底用 zai；见 6.2.1。
- **若想成本分层**：可把 brainstorm 换成 `qwen-portal/coder-model`（OAuth 免 Key）；writer/coding 按上表选。
- **密钥**：通义用 OAuth（`openclaw models auth login --provider qwen-portal`）；智谱/DeepSeek/MiniMax 等在 `.env` 里配对应 `*_API_KEY`，Runbook/ClawCN 国产模型指南有完整字段名。

### 6.4 小结

- M2 已落地：两群（brainstorm + coding）+ 两模型（DeepSeek + 智谱）+ 两套 SOUL；main 兜底。加新群时改 config、写新 workspace 的 SOUL，再 `openclaw gateway restart` 并同步 `.env`。
- 划分按「脑暴 / 写作 / coding / main（+ 可选资讯）」；国产模型优先 DeepSeek、智谱、通义、MiniMax，按「主力 / 省成本 / 长文 / 代码」分层分配（当前实现见 6.2.1）。
